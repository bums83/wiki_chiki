# OpenAI Privacy Filter

`OpenAI Privacy Filter` — это open-source модель для обнаружения и маскировки PII (Personally Identifiable Information) в тексте.

OpenAI выложила модель на Hugging Face, сделав её доступной для локального развёртывания и интеграции в собственные пайплайны обработки данных.

## Что она умеет

Классифицирует каждый токен текста по категориям PII:
- имена
- адреса
- email
- телефоны
- URL
- даты
- номера счетов
- API-ключи и токены

После классификации модель может:
- замаскировать чувствительные данные
- развернуть маскировку обратно (при необходимости)

## Практический смысл

Privacy Filter полезен в нескольких сценариях.

### 1. Подготовка данных для LLM-интеграций

Когда данные отправляются во внешние LLM-сервисы, PII может стать проблемой как с точки зрения compliance, так и с точки зрения безопасности. Privacy Filter позволяет заранее очистить текст от чувствительных данных перед отправкой.

### 2. Локальная обработка чувствительных данных

В сценариях, где данные не должны покидать локальное окружение, Privacy Filter можно использовать как часть пайплайна предобработки. Это особенно актуально для:
- медицинских записей
- финансовых документов
- HR-данных
- клиентской поддержки

### 3. Аудит и compliance

Модель может служить инструментом для регулярного аудита текстовых данных на наличие PII, что важно для соответствия GDPR, HIPAA и другим регуляторным требованиям.

## Ограничения

Как и любая модель для классификации текста, Privacy Filter может иметь:
- false positives (ложные срабатывания)
- false negatives (пропущенные PII)
- ограничения по языкам и форматам данных

Для критичных сценариев рекомендуется:
- тестировать модель на реальных данных
- настраивать пороги срабатывания
- комбинировать с другими методами защиты данных
