# VoxCPM2 Portable

`VoxCPM2 Portable` — это portable Windows-обвязка вокруг open-source модели `VoxCPM2` от OpenBMB.

Она превращает сырую модель в локальный прикладной инструмент: установка в один клик, `Gradio`-интерфейс, преднастроенный стек зависимостей и встроенный пайплайн обучения `LoRA` из длинного видео или аудио.

## Что именно делает

Проект объединяет четыре режима в одном интерфейсе:
- обычный `text-to-speech`,
- `Voice Design`, то есть создание нового голоса по текстовому описанию,
- `Voice Cloning` по короткому референсу,
- `LoRA`-дообучение собственного голоса.

По README заявлены:
- 30 языков с автоопределением,
- выход до `48 kHz` через `AudioVAE V2`,
- live-streaming воспроизведение во время генерации,
- набор готовых голосов,
- экспорт в `MP3`, `WAV`, `FLAC`, `OGG`.

## Чем portable-обвязка отличается от «просто модели»

| Слой | Оригинальный VoxCPM2 | VoxCPM2 Portable |
|------|-----------------------|------------------|
| Модель | Базовая TTS-модель | Та же модель, но упакованная для конечного пользователя |
| Установка | Требует ручной сборки окружения | `install.bat` ставит portable Python, PyTorch, ускорители и FFmpeg |
| Интерфейс | Нужен CLI или собственная обвязка | Готовая `Gradio`-студия |
| Fine-tuning | Нужно собирать pipeline отдельно | Есть вкладка `LoRA` и авто-подготовка датасета |
| Целевая аудитория | Разработчики и исследователи | Пользователь Windows, которому нужен рабочий desktop tool |

Практический смысл ровно в этом: проект не создаёт новую модель, а резко снижает порог входа.

Это близко по духу к [Вайб-кодинг]({{ '/wiki/llm-agents/vibe-coding' | relative_url }}): сложная система становится не research-артефактом, а инструментом, который можно быстро запустить и проверить в работе.

## Как устроен pipeline обучения LoRA

Самая сильная часть проекта — не обычный TTS, а автоматизация сбора датасета и запуска тренировки.

Заявленный цикл такой:
1. пользователь загружает длинное видео или аудио,
2. `ffmpeg` вытаскивает звук и приводит его к рабочему формату,
3. `Parakeet TDT 0.6B v3` делает `ASR`,
4. `Silero VAD` помогает отделять голосовые фрагменты,
5. текст и таймкоды собираются в клипы и транскрипты,
6. приложение подбирает параметры `LoRA`,
7. обучение можно стартовать в том же UI без второго контура.

Это важнее обычного клонирования по одной фразе, потому что проект пытается закрыть весь путь от сырого медиафайла до обученного голосового адаптера.

## Требования и ограничения

Проект ориентирован на `Windows 10/11` и `NVIDIA GPU`.

Заявленный минимум:
- `8 GB VRAM`,
- `16 GB RAM`,
- около `15 GB` диска.

Практически разумнее считать рабочим режимом:
- `12+ GB VRAM`,
- `32 GB RAM`,
- `30 GB` диска, если хранить модели, голосовые паки и `LoRA`-чекпоинты.

CPU-only режим заявлен, но сам источник предупреждает, что он будет очень медленным.

То есть это не универсальный «запускается везде» инструмент, а скорее desktop pipeline под локальную Windows-машину с нормальной видеокартой.

## Что в проекте выглядит сильным

- local-first модель, без облака и API-ключей,
- один UI для синтеза, дизайна голоса, клонирования и обучения,
- автоматическая подготовка датасета,
- hot-swap `LoRA` без полного перезапуска,
- сборка зависимостей внутрь portable-папки.

Это хороший пример того, как сложный мультимедийный стек упаковывают в прикладной продукт.

В этом смысле проект перекликается с [HyperFrames]({{ '/wiki/llm-agents/hyperframes' | relative_url }}): ценность не только в модели как таковой, а в интерфейсе и пайплайне, которые делают технологию операбельной.

## Где стоит сохранять трезвость

У проекта есть и слабые места.

Во-первых, источник явно маркетинговый и местами перегретый по тону.

Во-вторых, в README есть численная нестыковка по времени обучения: для примерно 8 минут речи в одном месте указано около `14 минут` на `RTX 4090`, а в другом месте для очень похожего объёма указано уже `3-4 минуты`.

Это не ломает сам проект, но означает, что заявленные тайминги надо считать ориентиром, а не гарантией.

В-третьих, почти вся ценность сборки завязана на Windows-окружение, portable-скрипты и совместимость с локальным железом. Для Linux или headless-сценариев статья README пользы почти не даёт.

## Практический вывод

`VoxCPM2 Portable` — это не новая голосовая модель, а упаковка модели и training-pipeline в usable desktop tool.

Если задача звучит как «хочу локально синтезировать, клонировать и быстро дообучить голос без ручной сборки стека», проект выглядит сильным.

Если задача звучит как «нужен воспроизводимый инженерный pipeline под сервер, Linux и automation-first окружение», на README пока видно меньше зрелости, чем на desktop-слое.

По operational-логике это можно поставить рядом с [Antfarm]({{ '/wiki/tools/antfarm' | relative_url }}): здесь сложный мультимедийный pipeline упакован в usable desktop workflow, а там многошаговая автоматизация упаковывается в устойчивый orchestration layer. Оба проекта ценны именно как упаковка сложного контура в рабочий инструмент.
