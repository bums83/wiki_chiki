# Оптимизация рабочего стола Windows

Стандартный интерфейс Windows 10/11 потребляет сотни мегабайт ОЗУ за счёт XAML-анимаций, прозрачности, теней и телеметрии. На слабом железе это критично.

## Подход: модульная замена компонентов

Вместо патча системных файлов — замена тяжёлых shell-компонентов на легковесные альтернативы, использующие классические Win32 API.

### Панель задач

**RetroBar** (.NET) — полноценная замена Taskbar через оригинальные API. Не нагружает GPU эффектами. Потребляет ~45 МБ.

**Retro-shell** (форк OpenShell) — меню Пуск в стиле Luna, включает классическую синюю панель с кнопками выключения.

### Системный трей

Проблема: при быстрой загрузке Windows утилиты не успевают встроиться в трей. Решение — VBS-оркестратор: задержка 20 секунд, затем последовательный запуск tray-утилит с микро-паузами.

Компоненты:
- **3RVX** — регулятор громкости
- **Network Activity Indicator** — индикатор сетевой активности

### Оформление окон

**DWMBlurglass** — убирает скруглённые углы Windows 10 и артефакты заголовков при использовании классических тем. Не для прозрачности — для устранения визуального мусора.

### Замена UWP-приложений

UWP-версии Paint и Калькулятора заменяются классическими Win32-версиями. UWP-калькулятор известен медленным запуском.

### Шрифты

Segoe UI → Tahoma 8pt через реестр. MenuShowDelay = 400 для олдскульной задержки меню.

### Скринсейверы

Классические OpenGL-скринсейверы (3D-трубы, лабиринт) не работают на современных драйверах — чёрный экран. Нужны исправленные .scr с поддержкой современных буферов кадра.

## Результаты

| Метрика | До | После |
|---------|----|-------|
| UI RAM | ~300-500 МБ | ~70 МБ |
| CPU idle | >0% | 0% |

## Применимость

Подход оправдан для:
- Продления жизни старого железа (2-4 ГБ ОЗУ)
- Рабочих станций, где UI — не приоритет
- Сценариев с жёсткими ограничениями по ресурсам

Ограничения:
- Требует ручной настройки и VBS-скриптов
- Часть утилит может вызывать ложные срабатывания антивирусов (UltraUXthemepatcher и аналоги лезут в системные ресурсы)
- Поддерживаемость при обновлениях Windows — под вопросом
