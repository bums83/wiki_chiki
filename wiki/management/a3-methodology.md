# Методология A3

Метод принятия решений через визуальный документ формата A3. Сториборд рассуждений, показывающий «как мы к этому пришли». Альтернатива «театру презентаций». Разработана в 1970-х для быстрой передачи информации на производствах Toyota.

## Структура A3

| Блок | Содержание | Обязательный |
|------|-----------|:---:|
| 0. Заголовок | Название + подписи ЛПР | Да |
| 1. Бэкграунд | Контекст, как вышли на проблему | Да |
| 2. Текущая ситуация | Что происходит сейчас | Да |
| 3. Цели и задачи | Что можно сделать прямо сейчас | Да |
| 4. Коренные причины | Почему возникла проблема | Да |
| 5. Контрмеры | Какие действия помогут | Да |
| 6. План внедрения | Как реализуем | Нет |
| 7. Результаты | Поддерживающие действия и итоги | Нет |

## Как создать A3

1. Разобраться в ситуации (не торопиться рисовать)
2. Изучить процесс (см. [Кайдзен](/wiki/management/kaizen/), [Диаграмма Исикавы](/wiki/management/ishikawa-diagram/))
3. Выделить важные моменты — проблемы, повторные точки, непонятные узлы
4. Определить круг интересантов
5. Нарисовать A3 (бумага, программы для визуализации)
6. Провести nemawashi (обсудить с командой)
7. Итерировать с п.2 пока все не скажут «ок»

## Бэкграунд

Вводная часть содержит:
- **Current Situation** — текущее положение
- **Ideal Situation** — идеальное положение
- **Ultimate Goal** — высшая цель
- Разница между IS и CS = зона роста

## Анализ

Не всё подряд, а только элемент с меткой проблемы + шаг до/шаг после. Инструменты: [Исикава](/wiki/management/ishikawa-diagram/), горизонтальное древо, любой метод визуализации.

## Выбор вектора

Нестандартные критерии: затраты, риски, профит. Сколько вложить, что потерять, есть ли смысл. Метрики не должны быть подтянуты под желаемый исход.

## Реальный пример

Март 2022: запуск мерча для инфлюенсерши, 9 месяцев задержки, три откладывания, бюджет освоен. A3 показал клиенту: бэкграунд → анализ → корневые причины → вектор (затраты/риски/профит). Апрув в тот же день.

## Связь с P3.Express

Грамотно составленный A3 закрывает почти всю фазу A1-A10 методологии P3.Express — документ для старта проекта в одном документе.

## 5S для IT

Визуализация в A3 связана с 5S — организацией рабочего пространства:

| Этап | IT-применение |
|------|---------------|
| Seiri | Скрипты по своим Git-репозиториям |
| Seiton | Docker-контейнеры на своих серверах |
| Seiso | Бэкапы, cron-очистка логов |
| Seiketsu | Все микросервисы описаны в Wiki |
| Shitsuke | Изменения → Git + Wiki, сборки задокументированы |
