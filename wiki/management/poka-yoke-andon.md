# Poka-yoke и Andon

Два механизма контроля качества из [[Toyota Production System|TPS]]: предотвращение ошибок и сигнализация о проблемах.

## Andon (Андон)

Система оповещения о проблемах на конвейере. Шнур над линией + контроллер → динамик, световой столб, дисплей.

### Как работает

1. Оператор видит проблему → тянет шнур
2. Сигнал транслируется на дисплей/динамик
3. Конвейер останавливается
4. Бригадир приходит, осматривает, принимает решение
5. Массовый дефект → полная остановка процесса

### IT-параллель

Вася запушил битую ссылку в webhook → Ваня залочил релиз → тимлид получил отбивку. Нюанс: у Васи могут быть права на снятие лока, нотификация может быть не настроена.

### Правило

Каждый сотрудник должен уметь определить проблему и знать, как сообщить. Права на окончательное решение — у наиболее опытных.

## Poka-yoke (защита от дурака)

Система, исключающая возможность ошибки. Не «ловит» ошибку, а делает её невозможной.

### Принципы

- Быстрая обратная связь при отклонении
- Проверяет ровно то, что нужно
- Простая реализация (механическая или программная)

### Примеры

| Область | Пример |
|---------|--------|
| Аппаратная | Ассиметричные разъёмы SATA, ОЗУ, NVMe |
| Софт | Формы валидации пароля |
| Код | Запрет на редактирование конфига с URL |
| IDE | Проверка кода перед коммитом |

## Karakuri (каракури)

Механическое устройство, выполняющее функцию за человека. Срабатывает по сигналу, не требует сложных действий. IT-параллель: скрипт, пересылающий сообщения из трекера в мессенджер.

## Внедрение

Человеческая психика инертна — люди продолжают делать «как привыкли». Это не лень, а естественный механизм.

Два фактора успеха:
- **Терпение** — привычка формируется со временем
- **Ценность** — сотрудник видит личную выгоду (меньше работать, меньше авралов)

Перебор с контролем вреден: километры инструкций → саботаж вместо осознанного выполнения.
