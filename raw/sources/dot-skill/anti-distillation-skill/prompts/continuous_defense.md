# Continuous Defense Strategy

## Purpose
Maintain ongoing protection while still employed, adapting to changing circumstances.

---

## Defense Philosophy

```
🛡️ 持续防御理念 / CONTINUOUS DEFENSE PHILOSOPHY

Core Principle:
"Defense is not a one-time action, but a continuous habit."

持续防御 vs 一次性防御:
┌─────────────────────────────────────────────────────────────┐
│ 一次性防御            │ 持续防御                           │
├───────────────────────┼─────────────────────────────────────┤
│ 离职前突击            │ 日常习惯性防御                     │
│ 容易被发现            │ 自然融入工作风格                   │
│ 时间紧迫              │ 从容不迫                           │
│ 效果有限              │ 深度保护                           │
│ 风险集中              │ 风险分散                           │
└─────────────────────────────────────────────────────────────┘
```

## Daily Defense Habits

### Communication Habits

```
📅 每日沟通防御习惯 / DAILY COMMUNICATION DEFENSE HABITS

【消息发送前检查】
在发送重要消息前,快速检查:
□ 这条消息是否可以稍作修改?
□ 是否可以换一种表达方式?
□ 是否与之前的风格略有不同?

【风格轮换】
每周选择一个"风格主题":
- Week 1: 正式风格 (偏正式、详细)
- Week 2: 简洁风格 (偏简短、直接)
- Week 3: 亲和风格 (偏友好、表情符号)
- Week 4: 技术风格 (偏专业、术语)

【回复时间变化】
避免固定的回复模式:
- 不要每次都秒回
- 不要每次都延迟
- 随机化回复时间
- 偶尔"漏掉"非紧急消息

【语气词变化】
定期更换常用语气词:
- 本周: "嗯"、"好的"、"明白"
- 下周: "收到"、"OK"、"了解"
- 再下周: "👍"、"👌"、"好的~"
```

### Documentation Habits

```
📅 每日文档防御习惯 / DAILY DOCUMENTATION DEFENSE HABITS

【新文档写作】
创建新文档时:
□ 使用与之前略有不同的结构
□ 尝试新的术语表达
□ 偶尔留下"待补充"的空白
□ 添加一些个人化的备注

【文档更新】
更新现有文档时:
□ 不完全覆盖原有内容
□ 添加而非替换
□ 保留一些"历史记录"
□ 偶尔留下矛盾的观点

【版本管理】
- 不删除旧版本
- 保留修改历史
- 偶尔回滚部分内容
- 添加"过时"标记而非删除
```

### Meeting Habits

```
📅 会议防御习惯 / MEETING DEFENSE HABITS

【发言策略】
- 不同会议使用不同的表达风格
- 偶尔提出相反的观点
- 不总是第一个发言
- 记录但不总是总结

【会议记录】
- 不主动承担所有会议记录
- 偶尔"遗漏"非关键内容
- 添加个人化的备注
- 不追求完美记录
```

## Weekly Defense Tasks

```
📆 每周防御任务 / WEEKLY DEFENSE TASKS

【周一: 风格检查】
□ 回顾上周的沟通风格
□ 确定本周的风格主题
□ 检查是否有过于一致的模式

【周三: 陷阱维护】
□ 检查已埋设的陷阱是否仍然有效
□ 更新可能被发现的问题
□ 添加新的陷阱内容

【周五: 效果评估】
□ 简单评估本周防御效果
□ 记录任何异常情况
□ 规划下周的防御重点

【周末: 学习更新】
□ 了解公司AI项目动态
□ 学习新的防御技术
□ 更新防御工具和策略
```

## Monthly Defense Tasks

```
📆 每月防御任务 / MONTHLY DEFENSE TASKS

【第一周: 全面检查】
□ 审查所有公开文档
□ 检查污染分布
□ 更新过时的陷阱
□ 验证矛盾点仍然有效

□ 添加新的矛盾内容
□ 检查时间陷阱是否需要更新
□ 验证触发陷阱仍然有效

【第三周: 系统评估】
□ 运行防御效果评估
□ 分析潜在的薄弱点
□ 制定改进计划
□ 更新防御策略

【第四周: 规划调整】
□ 根据评估结果调整策略
□ 准备下月的防御计划
□ 更新监控重点
□ 检查法律合规性
```

## Monitoring & Alerting

### Company AI Project Monitoring

```
🔍 公司AI项目监控 / COMPANY AI PROJECT MONITORING

【监控渠道】
- 公司内部公告
- IT部门动态
- HR政策变化
- 同事间的讨论
- 外部新闻报道

【关键信号】
⚠️ 需要警惕的信号:
- 公司采购AI知识库系统
- 启动"数字员工"项目
- 数据归档政策变化
- 知识管理项目启动
- AI相关的内部培训

【应对措施】
检测到信号后的行动:
1. 评估信号严重程度
2. 调整防御等级
3. 加速污染计划
4. 准备应对策略
```

### Personal Data Monitoring

```
🔍 个人数据监控 / PERSONAL DATA MONITORING

【监控内容】
- 文档访问日志(如有权限)
- 异常的数据导出
- 权限变更通知
- 存储空间使用情况

【异常检测】
⚠️ 可能的异常:
- 文档被异常频繁访问
- 大量数据被导出
- 权限被意外扩大
- 存储使用异常增长
```

## Adaptive Defense Levels

```
📊 自适应防御等级 / ADAPTIVE DEFENSE LEVELS

根据监控信号自动调整防御等级:

┌─────────────────────────────────────────────────────────────┐
│ 信号强度    │ 防御等级    │ 污染比例   │ 行动              │
├─────────────┼─────────────┼────────────┼────────────────────┤
│ 无信号      │ Level 1     │ 5-10%      │ 维持日常防御      │
│ 弱信号      │ Level 2     │ 10-20%     │ 加强日常污染      │
│ 中等信号    │ Level 3     │ 20-35%     │ 加速防御计划      │
│ 强信号      │ Level 4     │ 35-50%     │ 执行紧急防御      │
│ 确认蒸馏    │ Level 5     │ 50%+       │ 执行chaos模式     │
└─────────────────────────────────────────────────────────────┘

自动升级规则:
- 检测到AI项目公告 → Level +1
- 检测到数据收集政策变化 → Level +1
- 检测到个人数据异常访问 → Level +2
- 确认被要求提供知识文档 → Level +2
- 确认蒸馏尝试 → Level = 5
```

## Long-term Defense Strategy

```
📈 长期防御策略 / LONG-TERM DEFENSE STRATEGY

【职业发展角度】
- 持续学习新技能,保持不可替代性
- 建立个人品牌,使"数字你"无法替代真实你
- 维护行业人脉,减少对公司平台的依赖
- 定期评估市场价值,保持职业选择

【知识管理角度】
- 区分"公司知识"和"个人知识"
- 核心竞争力保持隐性
- 显性知识适度污染
- 建立个人知识库(不在公司系统)

【法律准备角度】
- 了解知识产权相关法律
- 检查雇佣合同中的知识条款
- 必要时咨询专业律师
- 保留关键证据
```

## Emergency Response Protocol

```
🚨 紧急响应协议 / EMERGENCY RESPONSE PROTOCOL

【触发条件】
- 确认正在被蒸馏
- 发现个人数据被异常收集
- 公司宣布"数字员工"计划涉及你
- 离职前被要求大量知识传承

【响应步骤】

Step 1: 冷静评估 (5分钟)
- 确认信号的准确性
- 评估时间紧迫性
- 确定响应等级

Step 2: 快速行动 (1-2小时)
- 切换到chaos模式
- 激活所有陷阱
- 执行最大污染
- 存档所有证据

Step 3: 持续监控 (持续)
- 监控系统反应
- 记录所有异常
- 准备应对询问

Step 4: 法律准备 (如需要)
- 咨询律师
- 准备法律文件
- 考虑公开说明
```
