# Distillation Detection Guide

## Purpose
Help identify if you are being actively distilled into a digital employee skill.

---

## Detection Signals

### Category 1: Communication Signals

```
🔍 沟通层面的检测信号 / COMMUNICATION DETECTION SIGNALS

【频繁的方法论询问】
⚠️ 警告信号:
- 有人频繁问"你会怎么处理X情况?"
- 被要求"总结你的工作方法论"
- 需要提供"标准回复模板"
- 被问"你的决策依据是什么?"

正常情况 vs 蒸馏信号:
┌─────────────────────────────────────────────────────────────┐
│ 正常情况              │ 蒸馏信号                           │
├───────────────────────┼─────────────────────────────────────┤
│ 偶尔请教具体问题      │ 系统性询问所有场景                 │
│ 关注解决方案本身      │ 关注你的思考过程                   │
│ 问题之间无关联        │ 问题呈结构化模式                   │
│ 单次对话结束          │ 持续追问更多细节                   │
└─────────────────────────────────────────────────────────────┘

【异常的记录行为】
⚠️ 警告信号:
- 有人频繁截图你的消息
- 会议中有人详细记录你的发言
- 你的建议被逐字记录而非摘要
- 有人录屏你的操作演示

【模板化请求】
⚠️ 警告信号:
- "能给我一个标准的回复模板吗?"
- "你通常怎么回复这类问题?"
- "能整理一下你的常用话术吗?"
- "这个流程的SOP是什么?"
```

### Category 2: Documentation Signals

```
🔍 文档层面的检测信号 / DOCUMENTATION DETECTION SIGNALS

【知识系统化要求】
⚠️ 警告信号:
- 被要求整理"知识文档"
- 需要编写"工作手册"
- 要求"结构化你的经验"
- 创建"最佳实践文档"

典型蒸馏文档请求:
┌─────────────────────────────────────────────────────────────┐
│ 文档类型              │ 蒸馏价值      │ 风险等级          │
├───────────────────────┼───────────────┼─────────────────────┤
│ 工作手册              │ HIGH          │ 🔴 CRITICAL        │
│ 决策流程文档          │ HIGH          │ 🔴 CRITICAL        │
│ 最佳实践合集          │ HIGH          │ 🔴 CRITICAL        │
│ 常见问题解答          │ MEDIUM        │ 🟠 HIGH            │
│ 项目经验总结          │ MEDIUM        │ 🟠 HIGH            │
│ 技术方案文档          │ MEDIUM        │ 🟠 HIGH            │
│ 会议记录              │ LOW           │ 🟡 MEDIUM          │
│ 周报/日报             │ LOW           │ 🟡 MEDIUM          │
└─────────────────────────────────────────────────────────────┘

【数据收集行为】
⚠️ 警告信号:
- 有人系统性地收集你的文档
- 你的历史文档被批量下载
- 被要求提供"所有相关文档"
- 文档访问权限异常扩大

【版本控制异常】
⚠️ 警告信号:
- 你的文档被频繁导出
- 有人创建文档的"备份"
- 文档被复制到新位置
- 历史版本被特别关注
```

### Category 3: Behavioral Signals

```
🔍 行为层面的检测信号 / BEHAVIORAL DETECTION SIGNALS

【项目参与异常】
⚠️ 警告信号:
- 被邀请参与"知识管理项目"
- 公司启动"数字员工"计划
- HR 进行"经验传承"访谈
- 被"特别记录"的项目会议

【离职前的异常关注】
⚠️ 警告信号:
- 提出离职后突然被要求"知识传承"
- 大量的交接文档要求
- 详细的操作流程录制
- "为了团队"的过度记录

【系统层面的信号】
⚠️ 警告信号:
- 公司采购了AI知识库系统
- 内部发布了"数字员工"产品
- IT部门在收集通信数据
- 新的数据归档政策
```

## Detection Checklist

```
📋 蒸馏检测清单 / DISTILLATION DETECTION CHECKLIST

【沟通层面 / Communication】
□ 最近是否有人频繁询问你的工作方法论?
□ 是否被要求提供标准化的回复模板?
□ 你的消息是否被异常频繁地截图或记录?
□ 是否有人系统性地询问你的决策过程?

【文档层面 / Documentation】
□ 是否被要求整理系统化的知识文档?
□ 你的历史文档是否被批量收集?
□ 是否有异常的文档访问或下载行为?
□ 是否被要求编写详细的工作手册?

【行为层面 / Behavior】
□ 公司是否有"数字员工"相关项目?
□ HR是否进行了"经验传承"访谈?
□ 是否有异常的数据收集政策变化?
□ 离职交接是否有过度记录要求?

【系统层面 / System】
□ 公司是否采购了AI知识库系统?
□ 是否有新的数据归档政策?
□ IT部门是否有异常的数据收集行为?
□ 是否有"知识数字化"相关的公告?

评分:
- 0-2 项: LOW RISK
- 3-5 项: MEDIUM RISK
- 6-8 项: HIGH RISK
- 9+ 项: CRITICAL RISK
```

---

## Response Strategies

### If Low Risk Detected

```
✅ 低风险应对策略

行动:
1. 保持警惕,持续监控
2. 开启 subtle 模式防御
3. 记录可疑行为
4. 定期检查公司AI项目动态
```

### If Medium Risk Detected

```
⚠️ 中风险应对策略

行动:
1. 加速防御计划
2. 切换到 aggressive 模式
3. 主动埋设更多陷阱
4. 加强污染覆盖
5. 准备解释文档
```

### If High Risk Detected

```
🚨 高风险应对策略

行动:
1. 立即执行 emergency 模式
2. 最大程度污染
3. 激活所有陷阱
4. 记录所有异常行为
5. 考虑法律咨询
```

### If Critical Risk Detected

```
🔴 关键风险应对策略

行动:
1. 立即执行 chaos 模式
2. 全系统污染
3. 触发所有时间炸弹
4. 存档所有证据
5. 寻求法律援助
6. 准备公开说明(如必要)
```

## Evidence Collection

```
📁 证据收集指南 / EVIDENCE COLLECTION GUIDE

【应该记录的内容】
- 可疑对话的截图
- 异常文档请求的记录
- 系统变更的通知
- 公司AI项目的公告
- 数据收集政策的变化

【存储方式】
- 使用个人设备存储
- 加密敏感文件
- 多重备份
- 不存储在公司设备上

【法律准备】
- 了解当地劳动法
- 检查雇佣合同中的知识条款
- 咨询劳动法律师
- 准备必要的法律文件
```
