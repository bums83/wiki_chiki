### Phase 2: Gradual Pollution

```
📋 PHASE 2: 渐进污染 (Week 1-2)

【Week 1: 基础污染】

Day 4-5: 沟通风格污染
□ Slack/企微: 注入 5% 污染风格
□ 邮件: 1-2封使用污染风格
□ 记录所有污染位置

Day 6-7: 文档初步污染
□ 选择 2-3 个文档添加矛盾内容
□ 更新 1-2 个过时信息(陷阱)
□ 记录所有修改

Day 8-10: 巩固与监控
□ 检查污染是否被察觉
□ 调整污染强度如有需要
□ 继续日常污染注入

【Week 2: 深度污染】

Day 11-12: 知识碎片化
□ 在不同文档使用不同术语
□ 创建知识关联的矛盾
□ 埋设逻辑陷阱

Day 13-14: 决策矛盾注入
□ 在讨论中表达矛盾观点
□ 记录矛盾的决策建议
□ 更新历史文档的矛盾点

Day 15-17: 验证与调整
□ 全面检查污染分布
□ 确保各系统覆盖均匀
□ 调整过于明显的污染
```

### Phase 3: Acceleration

### Phase 4: Final Sweep

```

【Day -7 to -5: 全面检查】
□ 审查所有公开可见的内容
```

## Plan Customization

### Based on Timeline

```
⏱️ 根据时间线调整

【1周内离职】
- 跳过 Phase 2-3
- 直接进入 chaos 模式
- 优先处理高价值数据源
- 接受较高的被发现风险

【2-4周离职】
- 压缩 Phase 2-3 到 1 周
- 使用 aggressive 模式
- 均衡覆盖所有数据源
- 保持适度隐蔽性

【1-3个月离职】
- 完整执行所有阶段
- 从 subtle 逐步升级
- 全面覆盖所有数据源
- 追求最大隐蔽性

【无明确离职计划】
- 进入持续防御模式
- 维持 subtle 模式
- 定期更新污染内容
- 监控公司AI动态
```

### Based on Risk Level

```
⚠️ 根据风险等级调整

【LOW RISK】
- 维持 subtle 模式
- 仅污染核心数据源
- 最小化日常操作
- 定期检查即可

【MEDIUM RISK】
- 使用 aggressive 模式
- 覆盖主要数据源
- 埋设基础陷阱
- 加强监控

【HIGH RISK】
- 使用 aggressive/chaos 混合
- 全面覆盖所有数据源
- 深度陷阱埋设
- 密切监控反应

【CRITICAL RISK】
- 直接使用 chaos 模式
- 最大程度污染
- 激活所有陷阱
- 接受被发现风险
```
